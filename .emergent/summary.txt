<analysis>**original_problem_statement:**
The user wants to build a production-ready, local-first “Darts Kiosk + Admin Control” system for a cafe. The system will run on a Mini-PC connected to each dartboard. A central admin panel, running on a MASTER PC, must be able to control all AGENT PCs on the local network.

**PRODUCT REQUIREMENTS:**
- **Core Functionality:** A kiosk UI that is locked by default. Staff can unlock it for customers to play. It auto-locks after the game ends or time/credits expire.
- **Architecture:** A MASTER/AGENT model for multi-board control over a local LAN. Each AGENT is autonomous if the MASTER is offline.
- **Roles:** Admin (full access: branding, pricing, management) and Staff (limited to session control: unlock/lock/extend).
- **Authentication:** JWT-based with username/password, plus a quick PIN action for staff. A Stammkunde (registered player) mode with nickname/PIN login is also required.
- **Customization (White-Label):** Admins can upload a logo, set the cafe name, and choose from pre-defined color palettes.
- **Pricing Models:** Configurable pricing per game, per time, or per player.
- **Autodarts Integration:** An MVP integration using Playwright to automate starting a game and detecting its completion on the Autodarts website.
- **Real-time & Discovery:** The system must use WebSockets for real-time status updates and mDNS for automatic discovery of agent PCs on the network.
- **Player Features:** Public QR-code links to match results, guest-based player statistics, and leaderboards.
- **Tech Stack:** FastAPI (Python) backend, React frontend, SQLAlchemy with SQLite.
- **Reliability:** The system must be robust, with heartbeats, offline capability, logging, a circuit breaker for the Autodarts integration, and automated database backups.
- **Deployment:** An appliance-like experience on Linux, with a one-command  script and a web-based setup wizard.
